1. 整体架构概览
================================================================================

技术栈
------
  后端：FastAPI + LangGraph + LangChain + SQLAlchemy
  前端：Vue 3 + Element Plus + ECharts
  数据库：SQLite（开发）/ MySQL（生产）
  缓存：Harness 内存缓存（SHA-256 哈希键）

Agent Harness 模式
------------------
  所有核心智能体通过 AgentHarness 统一管理：
    harness.run(agent_id, input_data, user_id)
  返回格式：
    {"success": true/false, "data": {...}, "cached": bool, "run_id": str, "duration_ms": int}

  注册流程：
    backend/app/agents/registry.py 中 import agent 并调用 harness.register(agent)

  智能体基类：AgentBase（backend/app/agents/base.py）
    - agent_id: str          智能体唯一标识
    - timeout_seconds: int   超时时间
    - cacheable: bool        是否缓存结果
    - build_graph() -> StateGraph   构建 LangGraph 图
    - run(input_data) -> dict       执行入口

2. 共享基础设施
================================================================================

2.1 LLM 工厂 — backend/app/agents/llm_factory.py
-------------------------------------------------
  get_llm(temperature, max_tokens) -> BaseChatModel
    - 使用 @lru_cache(maxsize=16) 缓存 LLM 实例
    - 模型：qwen-plus（阿里通义千问）
    - 基础 URL：https://dashscope.aliyuncs.com/compatible-mode/v1
    - 环境变量：DASHSCOPE_API_KEY

2.2 重试工具 — backend/app/agents/retry.py
------------------------------------------
  llm_call_with_retry(messages, temperature, max_tokens, max_retries, timeout)
    - 指数退避：min(2^attempt, 8) 秒
    - 每次调用超时：asyncio.wait_for(timeout)
    - 总尝试次数：max_retries + 1

  parse_json_response(content)
    - 去除 ```json / ``` 标记
    - json.loads 解析

2.3 Harness — backend/app/agents/harness.py
-------------------------------------------
  AgentHarness 单例
    - register(agent)          注册智能体
    - run(agent_id, input, user_id)  执行智能体
    - 内存缓存：SHA-256(input_json) -> result
    - 重试：指数退避 min(2^attempt, 8)
    - 超时：agent.timeout_seconds

3.1 简历分析智能体 — resume_analyzer
----------------------------------------------------------------------
  目录：backend/app/agents/resume_analyzer/
  文件：state.py, prompts.py, nodes.py, graph.py, tools.py, __init__.py

  agent_id     : "resume_analyzer"
  超时         : 120 秒
  缓存         : 是
  LLM 调用次数  : 1-2 次
  温度         : 0.1

  图流程：
    parse_file ──> analyze_resume ──> END

  节点说明：
    parse_file       解析上传的简历文件（PDF/Word/TXT），提取纯文本
    analyze_resume   调用 LLM 分析简历，提取结构化信息

  输入：
    file_path   简历文件路径
    file_type   文件类型（pdf/docx/txt）

  输出：
    resume_data  结构化简历数据（姓名、学校、技能、经历等）

  Prompt 核心要求：
    - 从简历中提取个人信息、教育背景、技能、项目经历、实习经历、证书
    - 输出纯 JSON 格式

3.2 岗位匹配智能体 — job_matcher
----------------------------------------------------------------------
  目录：backend/app/agents/job_matcher/
  文件：state.py, prompts.py, nodes.py, graph.py, tools.py, db_utils.py, __init__.py

  agent_id     : "job_matcher"
  超时         : 180 秒
  缓存         : 是
  LLM 调用次数  : 2-3 次
  温度         : 0.2

  图流程：
    load_profile ──> fetch_jobs ──> match_jobs ──> save_results ──> END

  节点说明：
    load_profile    加载用户画像（雷达图数据 + 维度详情）
    fetch_jobs      从数据库获取岗位列表
    match_jobs      调用 LLM 匹配岗位，计算匹配度分数
    save_results    保存匹配结果到 matching_report 表

  子模块：job_profiler.py（详见第4节）

  输入：
    user_id        用户 ID
    user_profile   前端传入的画像数据（可选）

  输出：
    matches        匹配岗位列表（排序后）
    top_job        最佳匹配岗位
    scores         7 维匹配分数

  匹配维度（7维）：
    专业技能、证书资质、创新能力、学习能力、抗压能力、沟通能力、实习/项目经验

  API 端点（backend/app/api/v1/matching.py）：
    POST /matching/match         执行匹配
    POST /matching/select-job    锁定岗位
    GET  /matching/selected-job  获取已锁定岗位
    GET  /matching/has-matching  检查是否有匹配数据
    GET  /matching/capability-model  获取能力模型数据

3.5 报告智能体 — report
----------------------------------------------------------------------
  目录：backend/app/agents/report/
  文件：state.py, prompts.py, nodes.py, graph.py, tools.py, __init__.py

  agent_id     : "report"
  超时         : 180 秒
  缓存         : 否（cacheable=False）
  LLM 调用次数  : 1-2 次
  温度         : 0.3

  图流程（generate）：
    load_all_data ──> generate_report ──> format_report ──> END

  图流程（polish）：
    load_all_data ──> polish_report ──> format_report ──> END

    user_id         用户 ID
    action          "generate" 或 "polish"

    report_text     报告文本

  报告结构：
    个人概况 → 能力分析 → 岗位匹配 → 学习计划 → 职业规划 → 总结建议

  数据库表：user_reports
    id, user_id (UNIQUE), report_text, created_at

  API 端点（backend/app/api/v1/report.py）：

----------------------------------------------------------------------
4.1 画像分析器 — profile_analyzer.py
----------------------------------------------------------------------
  位置：backend/app/agents/learning_plan/profile_analyzer.py
  调用方：learning_plan 智能体 + learning API

  核心函数：
    analyze_profile(chat_history, previous_radar_data, previous_details)
      -> {"radar_data": [7个int], "dimension_details": {7个维度}}

  处理流程：
    1. LLM 提取信息（EXTRACT_INFO_PROMPT）
       - 从对话历史中提取 7 维度相关信息
       - 温度 0.3，max_tokens 800
    2. LLM 评分（SCORE_DIMENSIONS_PROMPT）
       - 对 6 个维度打分（证书由算法评分）
       - 温度 0，max_tokens 1000
    3. 证书算法评分
       - 按领域分组，同领域取最高分，跨领域累加
       - 支持：英语、计算机等级、软考、云计算、网络、安全、Linux、AI、项目管理、竞赛
    4. 基准分兜底
       - 根据关键词检测确保最低分数
    5. 维度关联推断
       - 高分维度自动提升关联的低分维度（如实习→抗压）
    6. EMA 平滑
       - 新分权重 0.3，减少分数波动

  评分维度：
    专业技能、创新能力、学习能力、实习能力、抗压能力、沟通能力、证书

4.2 任务规划器 — task_planner.py
----------------------------------------------------------------------
  位置：backend/app/agents/learning_plan/task_planner.py

4.3 岗位画像器 — job_profiler.py

6. 跨智能体数据流
================================================================================

用户上传简历
    │
    ▼
resume_analyzer ──> 用户画像（7维分数 + 维度详情）
    │
    ├──> job_matcher ──> 匹配结果（岗位列表 + 匹配度）
    │         │
    │         ├──> [select-job] ──> 锁定岗位
    │         │         │
    │         │         ├──> career_planner ──> 职业规划
    │         │         │
    │         │         └──> learning_plan ──> 学习计划 + 每日任务
    │         │
    │         └──> matching_report 表
    │
    └──> diagnosis API ──> 诊断报告（PersonalInfo 页面）

报告智能体（report）聚合所有数据：
    用户画像 + 匹配结果 + 学习计划 + 职业规划 ──> 完整报告

触发关系：
    1. 用户上传简历 → resume_analyzer 自动触发
    2. 用户查看匹配 → job_matcher 触发
    3. 用户锁定岗位 → career_planner + learning_plan 异步触发
    4. 用户查看个人报告 → diagnosis API 触发
    5. 用户点击生成报告 → report 智能体触发

8. 缓存与容错策略
================================================================================

8.1 Harness 缓存
    - 键：SHA-256(input_json)
    - 存储：内存 dict
    - 失效：进程重启
    - 跳过：cacheable=False 的智能体（如 report）

8.2 重试策略
    - 所有 LLM 调用：指数退避 min(2^attempt, 8) 秒
    - 最大重试：2 次（总 3 次尝试）
    - 单次超时：30 秒（可配置）

8.3 前端缓存
    - PersonalInfo.vue：模块级变量缓存（SPA 内有效）
    - 匹配结果：sessionStorage 缓存
    - 雷达图数据：profileState.js 响应式状态

8.4 容错降级
    - LLM 调用失败 → 使用本地算法兜底（如 diagnosis 的 fallback 报告）
    - 证书评分 → 纯算法，不依赖 LLM
    - 基准分兜底 → 关键词检测确保最低分数

10. 优化建议
================================================================================

已完成的优化：
  [✓] career_planner 中 llm.invoke → llm.ainvoke（避免阻塞事件循环）
  [✓] job_profiler 和 profile_analyzer 统一使用 retry 工具
  [✓] harness 添加指数退避
  [✓] llm_factory 缓存大小 4 → 16
  [✓] matching API 添加异步任务错误回调
  [✓] report agent cacheable=False 避免缓存空结果
  [✓] PersonalInfo 加载动画升级为 InteractiveLoading

待优化：
  [ ] 岗位数据持久化 — 当前岗位列表硬编码在 tools.py 中，应迁移到数据库
  [ ] 用户认证完善 — 当前使用简单的 user_id，应实现 JWT 认证
  [ ] 文件上传清理 — 上传的简历文件缺乏定期清理机制
  [ ] LLM 调用监控 — 缺少调用次数、耗时、错误率的监控
  [ ] 前端错误处理 — 部分 API 调用缺少错误提示
  [ ] 数据库迁移 — 当前使用 init_db.sql 直接建表，应引入 Alembic 迁移
  [ ] 报告导出美化 — PDF 导出的中文字体支持和排版可进一步优化
